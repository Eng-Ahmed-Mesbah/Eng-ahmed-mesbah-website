About the Project

This was a university project where we had to design a hospital reception system using UML diagrams. The project was about creating a system that manages patient registration, appointments, medical tests, billing and other hospital operations.

We worked as a team of 5 engineers and we used Visual Paradigm software to make the UML diagrams. The project took us about 3 weeks to complete. It was a good learning experience because we learned how to design systems before coding them.

What the system does:

  • Patient can schedule appointments
  • Patient can pay bills
  • Patient can be admitted to hospital
  • Receptionist collects patient information
  • Receptionist provides receipts
  • Receptionist can assign beds for inpatients
  • Doctor can record medical reports
  • Doctor can request lab tests
  • Doctor can request radiology tests
  • Lab technician performs tests and sends results
  • Radiology technician performs tests and sends reports
  • Insurance company can file claims

What I learned

From this project I learned how to use UML diagrams to design systems. I also learned about the different types of relationships in UML like include and extend. Working with a team was also good because we had to coordinate and divide the work between us.

The most important thing I learned is that you need to plan and design before you start coding anything. The UML diagrams help developers understand what needs to be built.
